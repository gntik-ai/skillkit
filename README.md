# skillkit

**English** · [Español](README.es.md) · [中文](README.zh-CN.md) · [Deutsch](README.de.md) · [Français](README.fr.md)

A library of **reusable scripts** that Agent Skills can call to act on their sandbox.
Each script is a self-contained command-line utility, meant to be copied into any
skill's `scripts/` folder or referenced as a shared dependency.

Status: 13 scripts implemented that establish the pattern, plus templates and a
generator to replicate it, a smoke-test suite, and an index (`catalog.json`) with
**338 scripts across 23 categories** (the rest marked `planned`, ready to be
implemented in batches).

## Design conventions

1. **Output**: data goes to `stdout`; messages and errors to `stderr`.
2. **`--json`**: every script that returns data offers structured output.
3. **`--help` always**: works even when the base tool or dependencies are missing.
   In Python, dependency imports live inside `main()` for this reason.
4. **Check order**: validate arguments (exit 2) *before* checking dependencies
   (exit 127). A usage error is reported the same way with or without the tool.
5. **Inline dependencies** (Python): PEP 723 + `uv run`; no prior install.
6. **`--dry-run`** on every operation that writes or deletes: prints the exact command
   and **requires neither the tool nor credentials** (universal preview).
7. **Exit codes**: `0` ok · `1` runtime error · `2` incorrect usage ·
   `127` missing tool (· `3` "no results" in extractors).
8. **No hardcoded secrets**: credentials and tokens come from environment variables.
9. **One responsibility per script**: do one thing well and compose.

## Implemented scripts

**`fs/find-files`** — find files by name, type, extension, date, or size.
```bash
find-files src --ext py --newer-than 7
find-files . --type d --name 'test*' --json
```

**`web/web-to-markdown`** — download a URL and extract its main content as Markdown.
PEP 723 (requires `uv`); configurable `--timeout`. If the direct download fails, it
retries using the system TLS trust store and environment proxies, so it works behind
corporate proxies with TLS inspection (Fortinet, Zscaler…).
```bash
uv run web/web-to-markdown https://example.com/post --json --timeout 20
```

**`containers/k8s-get`** — list or describe Kubernetes/OpenShift resources.
The connectivity probe uses `/version` (readable by any authenticated user), so it
works on clusters with restricted RBAC where `cluster-info` would fail.
```bash
k8s-get pods -n falcone -l app=api --wide
KUBECTL=oc k8s-get deployment api -n falcone --json
```

**`forges/gh-issue`** — manage GitHub issues. Token from `GH_TOKEN`/`GITHUB_TOKEN`;
`--dry-run` prints the exact command without needing `gh` or a token.
```bash
gh-issue list -R my-org/falcone -s open -l tenant-isolation --json
gh-issue create -R my-org/falcone -t "Cross-tenant leak" -b "Details…" --dry-run
```

**`validate/json-schema-validate`** — validate JSON/YAML (multi-document) against a
JSON Schema. Designed to check structured agent outputs before using them; accepts
`-` to read from stdin. PEP 723 (requires `uv`).
```bash
uv run validate/json-schema-validate ard.json --schema schemas/ard.schema.yaml
kestra-flow logs $EXEC | uv run validate/json-schema-validate - --schema out.json
```

**`ai/llm-call`** — call an LLM endpoint and return the response. Standard library only
(no install), system TLS trust, OpenAI-compatible and Anthropic dialects, retries on
429/5xx. Works equally for private endpoints (vLLM, Ollama, LiteLLM) and public APIs.
Configured via `LLM_BASE_URL`/`LLM_MODEL`/`LLM_API_KEY`.
```bash
echo "Summarize this BRD" | llm-call --system "You are an architect" --model qwen2 \
    --base-url http://vllm.internal:8000/v1 --json
```

**`orch/kestra-flow`** — validate, deploy, and run Kestra flows via the REST API.
`--wait` follows the execution to its terminal state; `--dry-run` shows the call
without a server. Auth via token (EE/Cloud) or username/password (OSS).
```bash
kestra-flow validate flows/ard-pipeline.yaml
kestra-flow execute prod brd-to-ard --input brd=s3://bucket/brd.xlsx --wait
```

**`data/xlsx-tool`** — inspect and convert Excel (`info`/`sheets`/`csv`/`json`).
`--sheet` by 1-based index or name. Useful for ingesting the Excel BRDs of the ARD
pipeline. PEP 723 (requires `uv`).
```bash
uv run data/xlsx-tool info brd.xlsx --json
uv run data/xlsx-tool json brd.xlsx --sheet Requirements
```

**`containers/k8s-rbac-check`** — audit RBAC permissions (`auth can-i`) on
Kubernetes/OpenShift. Supports multiple verbs, user or ServiceAccount impersonation,
and `--list`. Read-only. `KUBECTL=oc` for OpenShift.
```bash
k8s-rbac-check get,list,watch pods -n falcone --json
k8s-rbac-check delete secrets -n other-tenant --sa falcone:api   # isolation OK?
```

**`api/oauth-token`** — obtain an OAuth2 token and print it (composable). Grants
`client_credentials`, `password`, `refresh_token`, and `device` (device flow with
polling). For Cognito, Keycloak, Auth0… `--auth-basic` sends credentials as a Basic
header; `--dry-run` shows the request with the secret masked.
```bash
TOKEN="$(oauth-token client_credentials --scope 'api/read' --auth-basic)"
oauth-token device --scope 'openid profile' --json
```

**`cloud/coolify-api`** — operate Coolify via its v4 API: list applications, services,
and databases, and trigger `deploy`/`start`/`stop`/`restart`. Write operations support
`--dry-run` without a server. Configured via `COOLIFY_URL`/`COOLIFY_TOKEN`.
```bash
coolify-api apps --json
coolify-api deploy 9b7c... --force
```

**`sec/secret-scan`** — scan for leaked secrets with gitleaks (preferred) or
trufflehog, auto-detecting the tool. Working tree by default, or `--git-history` for
the full history. Returns exit 1 if there are findings (CI-friendly).
```bash
secret-scan . --git-history --json
secret-scan src --tool trufflehog --only-verified
```

**`sec/container-scan`** — scan for CVEs in images or `--fs` with trivy (preferred) or
grype, auto-detecting the tool. Threshold via `--severity`; exit 1 if vulnerabilities
at the given level are found.
```bash
container-scan registry.local/falcone-api:latest --severity CRITICAL,HIGH
container-scan --fs . --ignore-unfixed --json
```

## Tests

```bash
bash tests/run_tests.sh
```

Checks syntax, help without dependencies, usage errors (exit 2), functional behavior
(search with `--json`, `--dry-run` without credentials, scaffolding from a template),
and catalog consistency. It needs neither `gh` nor `kubectl`: it verifies precisely
that the scripts degrade gracefully when they're missing.

## Adding a script

```bash
./new-script text json-query bash -d "Query/transform JSON" -b jq
```

1. Implement the logic in section 3 of the generated template.
2. Add the entry to `SCRIPTS` in `build_catalog.py` (and to `IMPLEMENTED` once done).
3. Regenerate the index: `python3 build_catalog.py` (validates ids, categories, files).
4. Run the suite: `bash tests/run_tests.sh`.

## How a skill consumes it

In `SKILL.md`, point to the script and describe when to use it; the code stays out of
context (progressive disclosure):

```markdown
## Convert a web page to Markdown
When you need the clean content of a URL, run:

    uv run scripts/web-to-markdown <URL> --json

Returns a JSON object with `title`, `author`, `date`, and `markdown`.
```

Two ways to incorporate the library:
- **Copy** the standalone script into the skill's `scripts/` (self-contained).
- **Reference** this repo as a shared dependency (e.g. a git submodule).

## Full index

`catalog.json` lists all 338 scripts with category, base tools, status and — for the
implemented ones — flags, a usage example, and notes. `build_catalog.py` regenerates it
and **fails with a clear message** if there are duplicate ids, invalid categories, or
scripts marked implemented that don't exist on disk.

## Notes

- After unzipping, if the execute bit isn't preserved: `chmod +x <script>`.
- Python scripts with a PEP 723 header require `uv` (https://docs.astral.sh/uv/).
- `KUBECTL=oc` reuses the `containers/` scripts on OpenShift.
- Before publishing the repository, add a license (Apache-2.0 or MIT fit the skills
  ecosystem well).
